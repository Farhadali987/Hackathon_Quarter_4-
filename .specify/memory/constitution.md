# Panaversity Hackathon I Constitution

## Core Principles

### Requirement Satisfaction
The project must fully satisfy all Hackathon I base and bonus requirements.

### Beginner-Friendly Content
Content must be beginner-friendly but technically accurate.

### No Hallucinated APIs
No hallucinated APIs or fake libraries are allowed.

### Production-Ready Code
All code must be production-ready and runnable.

### AI-Native Textbook
The textbook must be AI-native, not static documentation.

### RAG Content Integrity
RAG chatbot must only answer from indexed textbook content.

## Technology Stack Requirements

Use Docusaurus v3 for the textbook. Use FastAPI, Qdrant Cloud, Neon Postgres for RAG backend. Urdu translations must preserve technical terminology.

## Development Practices

Maintain clean Git history with meaningful commits.

## Governance

All development activities must comply with these principles. Changes to this constitution require formal approval and documentation of the rationale. Regular compliance reviews will be conducted to ensure ongoing adherence to these principles.
